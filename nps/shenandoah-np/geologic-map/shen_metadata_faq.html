What does this data set describe?

Title:
Unpublished Digital Geologic Map of Shenandoah National Park and Vicinity, Virginia (NPS, GRD, GRI, SHEN, SHEN digital map) adpated from the U.S. Geological Survey Open-File map by Southworth, Aleinikoff, Bailey, Burton, Crider, Hackley, Smoot and Tollo (2009)
Abstract:
The Unpublished Digital Geologic Map of Shenandoah National Park and Vicinity, Virginia is composed of GIS data layers complete with ArcMap 9.3 layer (.LYR) files, two ancillary GIS tables, a Map PDF document with ancillary map text, figures and tables, a FGDC metadata record and a 9.3 ArcMap (.MXD) Document that displays the digital map in 9.3 ArcGIS. The data is also available as a 2.2 KMZ/KML file for use in Google Earth. Google Earth software is available for free at: <http://www.google.com/earth/index.html>. Users are encouraged to only use the Google Earth data for basic visualization, and to use the GIS data for any type of data analysis or investigation. The data were completed as a component of the Geologic Resources Inventory (GRI) program, a National Park Service (NPS) Inventory and Monitoring (I&M) funded program that is administered by the NPS Geologic Resources Division (GRD). Source geologic maps and data used to complete this GRI digital dataset were provided by the following: U.S. Geological Survey. Detailed information concerning the sources used and their contribution the GRI product are listed in the Source Citation sections(s) of this metadata record (shen_metadata.txt; available at <http://nrdata.nps.gov/shen/nrdata/geology/gis/shen_metadata.xml>). Users of this data are cautioned about the locational accuracy of features within this dataset. Based on the source map scale of 1:24,000 and United States National Map Accuracy Standards features are within (horizontally) 50.8 meters or 166.7 feet of their actual location as presented by this dataset. Users of this data should thus not assume the location of features is exactly where they are portrayed in Google Earth, ArcGIS or other software used to display this dataset. All GIS and ancillary tables were produced as per the NPS GRI Geology-GIS Geodatabase Data Model v. 2.1. (available at: <http://science.nature.nps.gov/im/inventory/geology/GeologyGISDataModel.cfm>). The GIS data is available as a 9.3 personal geodatabase (shen_geology.mdb), and as shapefile (.SHP) and DBASEIV (.DBF) table files. The GIS data projection is NAD83, UTM Zone 17N, however, for the KML/KMZ format the data is projected upon export to WGS84 Geographic, the native coordinate system used by Google Earth. That data is within the area of interest of Shenandoah National Park.
Supplemental_Information:
The data layers (feature classes) that comprise the Unpublished Digital Geologic Map of Shenandoah National Park and Vicinity, Virginia include: SHENGLG (Bedrock Units), SHENGLGA (Bedrock Contacts), SHENGSL (Geologic Sample Localities), SHENGPF (Geologic Point Features), SHENGOL (Geologic Observation Localities), SHENSUR (Surficial Units), SHENSURA (Surficial Contacts), SHENATD (Geologic Attitude Observation Localities), SHENDEF (Deformation Areas), SHENDEFA (Deformation Area Boundaries), SHENHZP (Hazard Point Features), SHENMIN (Mine Point Features), SHENHZA (Hazard Area Features), SHENHZAA (Hazard Area Feature Boundaries), SHENSEC (Geologic Cross Section Lines) and SHENFLT (Faults). There are three additional ancillary map components, the Geologic Unit Information (SHENUNIT) Table, the Source Map Information (SHENMAP) Table and the Map PDF Document (shen_geology.pdf). Refer to the NPS GRI Geology-GIS Geodatabase Data Model v. 2.1 (available at: <http://science.nature.nps.gov/im/inventory/geology/GeologyGISDataModel.cfm>) for detailed data layer (feature class) and table specifications including attribute field parameters, definitions and domains, and implemented topology rules and relationship classes. For the KML/KMZ format all or only some of the data layers are available. The KMZ file also possesses on-line links to the GRI program and its products, and to this readme document, FAQ metadata (in .html format), and the GRI ancillary map information (.pdf) document pertaining to this dataset.

  2. How were the data generated, processed, and modified?

    Date: 04-Nov-2009 (process 1 of 1)
    1.) GIS features were produced from source digital data. See the Source Information Contribution section(s) for specific source map details. GIS features were imported into a geodatabase as per the GRI data model. Topology rules were then imposed and validated, as per the GRI data model, to check for and correct topology errors. Quality control (QC) consisting of a visual check and/or queries of the data against its source (where applicable), as well as running a GRI developed script (QC_Script) to check for GRI data model validation and feature-related consistency, was conducted. 2.) GIS table attribution was derived and checked using the source digital data file(s). Relationship classes were also used to ensure attribution consistency between feature class and table attribution. For details on the GRI data model see the NPS GRI Geology-GIS Geodatabase Data Model v. 2.1 (available at: <http://science.nature.nps.gov/im/inventory/geology/GeologyGISDataModel.cfm>). 3.) GIS feature classes were then exported from the geodatabase to shapefile (.SHP) format, and GIS tables exported from the geodatabase to DBASEIV (.DBF) format. 4.) The map-related pdf document, see the Supplemental Information section for additional information, was produced from textual information and figures present on the source map(s) and/or in digital data files. If applicable, source map images were produced at 150dpi or greater resolution. The text, source map images and other ancillary source map information were added to a Help & Manual (.HMX) template file. The Help & Manual (.HMX) file was then compiled to produce the .PDF map document. Any compilation errors were then checked and corrected and the document was reviewed for content, usability and grammatical errors. KML/KMZ files were then produced with key table fields appended to each data layer prior to export.
